# VibeApp 产品风格

## 产品定位

**一句话**：把截图变成可回顾的洞察。

**核心价值**：从相册截图中提取文字、整理信息，并按时间线复盘与总结。帮助用户从碎片截图中获得连贯洞察。

## 功能优先级

1. **截图导入与 OCR**：本地 OCR，快速生成卡片
2. **结构化整理**：来源、标签、摘要、关键词
3. **时间线复盘**：按时间浏览、搜索、筛选
4. **AI 趋势分析**：云端仅处理文本，返回摘要、相似点、趋势
5. **登录与同步**：可选，不强制

## 文案基调

### 语气
- **简洁直接**：少形容词，多用动词和名词
- **引导式**：告诉用户「可以做什么」「下一步是什么」
- **友好但专业**：不卖萌，不啰嗦

### 示例

| 场景 | ✅ 推荐 | ❌ 避免 |
|------|---------|---------|
| 首页副标题 | 把截图变成可回顾的洞察。 | 快来体验智能截图管理吧！ |
| 空状态 | 还没有内容。先导入几张截图开始。 | 空空如也，快去添加吧～ |
| 空状态引导 | 导入截图后自动生成卡片 | 点击下方按钮添加你的第一张截图 |
| 错误提示 | 识别失败：\(error) | 糟糕，出了点问题 |
| 状态说明 | 已就绪，等待导入截图。 | 一切准备就绪！ |

### 功能命名
- **导入截图**（不用「添加」「选择」）
- **生成趋势总结**（不用「一键分析」「AI 总结」）
- **历史记录** / **时间线**
- **趋势分析** / **趋势报告**
- **我的账号**（不用「个人中心」）

## UX 原则

### 本地优先
- 截图、OCR 均在本地完成
- 云端仅接收文本与必要元数据，不上传图片（除非用户显式开启）
- 文案中强调「本地」「本机」以建立信任

### 渐进式复杂度
- 首次使用：选图 → OCR → 看到卡片，零配置
- 进阶：补充来源、标签、摘要
- 可选：API Key、iCloud 同步、趋势分析

### 反馈与状态
- 处理中：明确文案（如「正在识别截图文字...」「正在生成趋势总结...」）
- 错误：简洁说明原因，可附带技术细节
- 成功：通过界面变化反馈，不弹 toast（除非必要）

### 表单与编辑
- 必填与选填通过 placeholder 和说明区分
- 标签、关键词用「逗号分隔」说明
- 敏感输入（API Key）提供显示/隐藏切换

## 非目标（MVP 及当前阶段）

- 不做复杂社交与分享
- 不强制上传图片
- 不做多语言自动翻译
- 不做复杂图表可视化

## 数据结构术语

| 概念 | 说明 |
|------|------|
| 截图 / 卡片 | 单条截图及其 OCR、标签、摘要等 |
| 时间线 | 按时间排列的卡片列表 |
| 趋势报告 | 一段时间内多张卡片的 AI 分析结果 |
| 标签 | 用户自定义分类（如学习、产品、购物） |
| 关键词 | 用于关联与检索的词 |

## 与 AI 协作时的注意

- 新增功能需符合「截图 → 整理 → 复盘 → 洞察」的主线
- 文案保持中文，与现有界面统一
- 涉及隐私时优先本地处理，云端操作需明确说明
